ATLAS V3.1  ·  D6  ·  REFERENCE-GRADE KIT

One kit. Five sections. Every rule inline.

The single component vocabulary shared across Floor, Bookings, Door, Closeout, Outcomes and Promoter surfaces. Each sample below is live and carries its usage rule as caption text — treat that caption as the component's acceptance spec. Dark-only, red-free, hub colors semantic. Money renders from integer cents; take-rate is a ⚑ placeholder.

A

Status system

DOCUMENTED FIRST

The ratified, red-free answer to "what color is this state?". Booking lifecycle chips plus the three-weight amber alarm ladder. This is the most load-bearing decision in the system — everything else defers to it.

Lifecycle status chips160ms color-only · position never moves
Held Confirmed Seated Good Closed Cancelled
SpecHELD = dashed MID chip + pulsing hollow dot (awaiting deposit). CONFIRMED = cyan. SEATED / GOOD = mint #34D399. CLOSED = gray. CANCELLED = gray with line-through. State changes animate color only over 160ms — the chip never changes position or size.
Amber alarm ladder — three weightsoutline -> tinted fill -> solid
△Attention ⊘Failed ⚑Blocking
SpecEscalate by weight, never by hue. W1 outline (attention) -> W2 tinted fill rgba(251,191,36,.12) (failed) -> W3 solid amber, dark text (blocking — max one per view). Negative money is amber and parenthesised.
NeverRed is never used — not for errors, declines, or destructive actions. Alarm severity is carried entirely by amber weight. This is the rule the whole system is built to protect.
